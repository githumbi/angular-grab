## Changes Made

### 1. Dead branch removed — `overlay-renderer.ts`

**File:** `packages/core/src/core/overlay/overlay-renderer.ts`

The `positionOverlay` function set `labelLeft` via `Math.max(0, ...)` then immediately checked `if (labelLeft < 0)`. Since `Math.max(0, x)` guarantees a non-negative result, the subsequent guard could never be true — it was unreachable dead code.

Removed:
```ts
if (labelLeft < 0) {
  label.style.left = '0px';
}
```

### 2. Duplicate sort+slice deduplicated — `mcp-server/src/index.ts`

**File:** `packages/mcp-server/src/index.ts`

The `angular_grab_recent` tool case manually sorted and sliced `history.entries` — exactly what `searchHistory()` does with no filter arguments. The duplication meant any future change to sort order or limit clamping in `searchHistory` would need to be mirrored here.

Before:
```ts
const sorted = [...history.entries].sort((a, b) => b.timestamp - a.timestamp);
const recent = sorted.slice(0, Math.max(0, limit));
```

After:
```ts
const recent = searchHistory(history, undefined, undefined, undefined, limit);
```

### 3. Dispose loop simplified — `plugin-registry.ts`

**File:** `packages/core/src/core/plugins/plugin-registry.ts`

The `dispose()` method iterated over `plugins` (to get names) then used those names to look up `cleanups`. Since `cleanups` is keyed the same as `plugins`, iterating `cleanups.values()` directly is simpler and removes the two-level lookup.

Before:
```ts
for (const [name] of plugins) {
  const cleanup = cleanups.get(name);
  if (cleanup) {
    try { cleanup(); } catch { /* ignore */ }
  }
}
```

After:
```ts
for (const cleanup of cleanups.values()) {
  try { cleanup(); } catch { /* ignore */ }
}
```

### 4. Duplicate `typeof` check removed — `esbuild-plugin/plugin.ts`

**File:** `packages/core/src/esbuild-plugin/plugin.ts`

The banner injection code called `typeof banner === 'object'` twice — once to extract `banner.js`, and again to spread the object. Extracted to a local variable to eliminate the repeated narrowing check.

Before:
```ts
const banner = build.initialOptions.banner || {};
const existing = typeof banner === 'object' ? (banner.js || '') : '';
build.initialOptions.banner = {
  ...(typeof banner === 'object' ? banner : {}),
  js: `...${existing}`,
};
```

After:
```ts
const existingBanner = build.initialOptions.banner;
const bannerObj = typeof existingBanner === 'object' ? existingBanner : {};
const existingJs = bannerObj.js || '';
build.initialOptions.banner = {
  ...bannerObj,
  js: `...${existingJs}`,
};
```

### 5. `any` replaced with typed interface — `keyboard-handler.ts`

**File:** `packages/core/src/core/keyboard/keyboard-handler.ts`

`navigator.userAgentData` is accessed via `(navigator as any)`. Added a minimal local interface for the non-standard API instead.

Before:
```ts
const uaData = (navigator as any).userAgentData;
```

After:
```ts
interface NavigatorUAData {
  platform: string;
}
const uaData = (navigator as Navigator & { userAgentData?: NavigatorUAData }).userAgentData;
```

### 6. `Set<any>` replaced with `Set<object>` — `component-resolver.ts`

**File:** `packages/core/src/angular/resolvers/component-resolver.ts`

The `seen` set tracks Angular component instances (guaranteed to be objects) for deduplication. `Set<object>` is accurate and avoids the loose `any`.

## Warnings

### Clipboard failure silently swallowed in history re-copy

**File:** `packages/core/src/core/grab.ts`, around line 366

```ts
} catch {
  // clipboard write failed silently
}
```

When a user clicks a history entry and clipboard write fails, no feedback is given. The toast that would show on success never fires. This leaves the user with no indication the action failed. A `showToast('Copy failed')` call in the catch block would close the gap.

Not fixed here because it requires deciding on the error message and could affect other error handling patterns across clipboard operations.

## Suggestions (not fixed)

### `init()` is a redundant wrapper

**File:** `packages/core/src/core/grab.ts`

`init()` is a single-line wrapper for `createGrabInstance()`. Both are exported. Consider dropping `init` from the public API in a future major version to reduce surface area.

### `context-builder.ts` selector differs from `copy.ts` selector

Two distinct element context builders exist:
- `buildElementContext` in `copy.ts` — builds a flat `tag#id.class` selector
- `buildContext` in `context-builder.ts` — builds a full CSS path (used in public API export)

They are intentionally different (copy vs. debug contexts), but the duplication should be documented so future contributors don't try to unify them and break the path-based selector.

### `angular-grab.service.ts` duplicates `createNoopApi`

The service has its own `createNoOpApi()` (note capitalization) that mirrors `createNoopApi()` in `grab.ts`. They can't be deduplicated without creating a circular dependency across the `core`/`angular` module boundary. Adding a comment noting this is intentional would help.

## Positive Patterns

- **Serialized write queue** in `mcp-server/index.ts` — `writeQueue` prevents concurrent read-modify-write races on the history file. Well-implemented.
- **All `console.log` uses stderr** in `mcp-server/index.ts` — all user-visible output uses `console.error`, correctly reserving stdout for MCP protocol messages.
- **Consistent dispose pattern** — every stateful module (`overlay-renderer`, `crosshair`, `freeze-overlay`, `toolbar-renderer`, etc.) implements a `dispose()` that removes DOM elements and unregisters listeners. No leaks observed.
- **Idempotent style injection** — every module guards `injectStyles()` with `if (document.getElementById(STYLE_ID)) return`. No duplicate style injection.
- **Proxy-based reactive store** — `createStore()` uses a `Proxy` to trigger listeners only when values actually change (`if (target[key] === value) return true`). Simple and correct.
- **Plugin error isolation** — `callHook` and `callTransformHook` wrap each plugin call in try/catch so a misbehaving plugin can't break the main flow.
- **Body size limit on webhook** — 512 KB cap with `req.destroy()` on overflow prevents OOM from malicious or accidental large payloads.
